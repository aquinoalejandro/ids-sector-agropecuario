Trabajo Práctico - Predicción de Heladas

Tema del Proyecto

Desarrollo de un modelo predictivo de heladas para la región de Córdoba, Argentina, utilizando registros históricos de variables meteorológicas horarias desde 1979.

Metodología

CRISP-DM (Cross-Industry Standard Process for Data Mining) - Un estándar de la industria para proyectos de minería de datos con enfoque sistemático.

Entregables
  • Esta notebook completa con análisis
  • Solución de software implementada
  • Modelos predictivos
Fuente de Datos

OpenWeather: Registros horarios de variables meteorológicas desde enero de 1979 hasta abril de 2020 para la ciudad de Córdoba.

Fase 1: Comprensión del Negocio

Problema de Negocio

El sector agropecuario de Córdoba enfrenta riesgos significativos por heladas inesperadas que pueden destruir en pocas horas toda una cosecha de frutas (duraznos, ciruelas) y hortalizas (tomates, pimientos, lechuga).

Solución propuesta: Modelo predictivo de heladas con 12-24 horas de anticipación para permitir medidas de protección.

Objetivo General

Desarrollar un modelo que prediga heladas con anticipación suficiente para que productores tomen medidas concretas de protección.

Objetivos Específicos
  • Explorar y preparar dataset meteorológico
  • Construir variable objetivo (helada/no helada)
  • Identificar variables predictivas
  • Entrenar modelos de clasificación
  • Validar con datos reales

Problemas y Desafíos Identificados
Estructura anidada: Campo 'weather' como lista de diccionarios que requiere normalización.
Timestamp como texto: dt_iso requiere conversión a datetime y ajuste a zona horaria UTC-3.
Datos faltantes masivos: >99% de nulos en rain, snow (se eliminan).
Variables constantes: lat, lon, city_name, timezone (sin variabilidad).
Desbalance de clases: Solo 0.48% de registros con helada.
Dependencia temporal: Observaciones consecutivas no son independientes.

Fase 3: Preparación de los Datos

Estrategia de Limpieza
1. Parseo de fechas

Conversión de dt_iso a datetime con zona horaria America/Argentina/Cordoba (UTC-3). Extracción de variables temporales: year, month, hour, season.

2. Eliminación de variables sin valor

Se eliminan: rain, snow, snow.1h, rain.3h, lat, lon, city_name, timezone, weather.icon, dt (reemplazado por dt_local).

3. Imputación de datos faltantes

rain.1h: Los valores NaN se reemplazan por 0, asumiendo que la ausencia de registro equivale a ausencia de lluvia.

4. Corrección de inconsistencias

Eliminación de registros con valores fuera de rangos físicos (<0.03% del dataset): presión <860 hPa, viento >360°, humedad fuera de 0-100%.

Variables Relevantes

Se mantienen variables con variabilidad temporal: temperatura, humedad, presión, viento, precipitación y cobertura de nubes.

Análisis Descriptivo

Estadísticas por variable, distribuciones, correlaciones y patrones temporales para identificar relaciones con heladas.

Fase 4: Modelado

Variable Objetivo

helada - Variable binaria que indica ocurrencia de heladas (1 si temperatura ≤ 0°C, 0 en caso contrario). Crucial para cumplir el objetivo de predecir heladas anticipadamente.

Random Forest

Algoritmo seleccionado para clasificación de heladas. Es robusto a relaciones no lineales y desbalance de clases.

Hiperparámetros

n_estimators=100, class_weight='balanced' para manejar el desbalance de clases (99.5% sin helada).

Razones para usar Random Forest
No Linealidad
Captura relaciones complejas basadas en umbrales (T ≤ 0°C)
Desbalance
Robusto ante 99.5% sin helada vs 0.5% con helada
Importancia
Proporciona ranking de qué variables importan más
Sobreajuste
Múltiples árboles reducen riesgo de overfitting
